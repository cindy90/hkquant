# R10-4: this project uses `uv` + `pyproject.toml`, NOT a plain
# requirements.txt. The file is preserved here only so legacy callers
# (e.g. CI pipelines that haven't migrated) get a clear error pointing
# at the correct install path.
#
# To install for development:
#     uv sync --extra dev --extra parse --extra reports
#
# To install production-minimal:
#     uv sync
#
# The dependency manifest itself lives in ``pyproject.toml`` under
# ``[project] dependencies`` / ``[project.optional-dependencies]``.
#
# The previous NACS-era pinned dependencies (pandas / openpyxl /
# akshare etc.) are now declared in pyproject.toml — pinning them again
# here would just cause version drift between the two manifests.
